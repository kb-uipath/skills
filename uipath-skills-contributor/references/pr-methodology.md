# Contribution Methodology

## Priority Rubric

Score candidates by impact per effort:

| Signal | Higher priority when |
|---|---|
| Agent/user impact | The bug sends an agent into a dead end, wrong command, unsafe auth path, invalid artifact, or repeated trial-and-error loop |
| Effort | The fix is docs, tests, small schema guidance, or a narrowly scoped helper change |
| Acceptance odds | The patch aligns with existing repo language, avoids product ownership questions, and has simple validation |
| Non-duplication | No open PR already addresses the issue or same files with same intent |
| Strategic fit | The change improves reliability, usability, or adoption in LLM/agentic automation workflows |

Prefer these surfaces:

- Wrong or stale command names, flags, or namespaces
- Malformed copy/paste snippets
- Missing guardrails that cause invalid generated projects
- Docs that are host-specific when they should be agent-neutral
- Clear issue-backed gaps in agent UX
- Small smoke tests when the repo already has a matching task folder and pattern

Deprioritize:

- Large product features requiring internal roadmap context
- Changes assigned to internal owners unless narrowly complementary
- Broad refactors of repo structure
- Unverifiable claims about product behavior
- Cosmetic wording without agent behavior impact

## Triage Commands

Useful starting points:

```bash
git status --short --branch
git fetch origin main
gh pr list --repo UiPath/skills --state open --limit 80 --json number,title,headRefName,author,url,labels,isDraft,mergeStateStatus
gh issue list --repo UiPath/skills --state open --limit 120 --json number,title,labels,assignees,url,updatedAt
```

For a candidate:

```bash
gh issue view <number> --repo UiPath/skills --comments --json number,title,state,assignees,labels,body,comments,url
gh pr list --repo UiPath/skills --state open --search "<keyword>" --json number,title,headRefName,url
```

## Branch And PR Conventions

- Start from `main` unless updating an existing PR branch.
- Use descriptive branch names such as `docs/rpa-agent-context-clarity` or `fix/platform-auth-guidance`.
- Use conventional PR titles that match repo style:
  - `docs(uipath-rpa): ...`
  - `fix(uipath-platform): ...`
  - `test(uipath-maestro-flow): ...`
- Do not include `[codex]`, "generated by Codex", or similar branding when the user asked not to.
- Do not label PRs unless the user or repo-specific instructions require labels.
